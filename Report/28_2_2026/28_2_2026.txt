The drop in performance on Grammar and Lexical Resource strongly suggests your model lacks the specific NLP features required to evaluate language mechanics. Here is how to fix that:

Since we are using traditional ML, we need to feed it highly specific linguistic features:
    For Grammar Range: Do not rely on raw text. Extract features using NLP libraries (like `spaCy` or `NLTK`). Count the number of dependent clauses, measure sentence length variance, and use a grammar-checking library (like `LanguageTool` API) to count the number of grammatical errors per 100 words.
    For Lexical Resource: Calculate the Type-Token Ratio (unique words divided by total words) to measure vocabulary diversity. Measure how many words belong to the Academic Word List (AWL). Count spelling errors.
    For Task Achievement: Use cosine similarity (via word embeddings) to measure how closely the essay's vocabulary aligns with the prompt's vocabulary.
